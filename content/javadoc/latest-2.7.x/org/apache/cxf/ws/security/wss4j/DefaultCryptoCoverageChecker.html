public class DefaultCryptoCoverageChecker
extends CryptoCoverageChecker

This utility extends the CryptoCoverageChecker to provide an easy way to check to see if the SOAP (1.1 + 1.2) Body was signed and/or encrypted, if the Timestamp was signed, and if the WS-Addressing ReplyTo and FaultTo headers were signed. The default configuration is that the SOAP Body, Timestamp must be signed, and WS-Addressing ReplyTo and FaultTo headers must be signed (if they exist in the message payload).
